# 学生信息查询系统 (SIS) 使用指南

## 🚀 **介绍**

欢迎使用学生信息查询系统（SIS）。本系统是一个功能强大的、基于网页的内部工具，旨在帮助授权用户高效查询学生的基本信息、班级归属、考试成绩等关键数据。

系统支持按姓名（或拼音缩写）和按特定代码（如学号、生日、地区等）两种查询模式。所有功能均受严格的认证机制保护，以确保学生数据的安全与合规。查询结果将以清晰、现代的卡片式布局呈现，并支持丰富的交互式操作以查看详情。

::: tip 重要提示
- 在您开始使用前，请务必仔细阅读本指南，以充分了解各项功能。
- 系统数据源自学校内部信息，仅供已获授权的用户出于教学和管理目的使用。请严格遵守隐私政策，不得将数据用于任何非法目的或泄露给无关人员。
- 如果您在访问或查询时遇到问题（如加载缓慢或无响应），请首先检查您的网络连接，或尝试刷新页面。
:::

## 🔐 **用户认证**

系统的一切访问权限均基于您的个人认证代码。

- **进入系统**：当您打开系统网页时，首先看到的将是认证界面。
- **输入认证代码**：您需要在系统提供的**认证代码框**中，逐格输入您的身份验证代码（即下文提到的32位MD5字符串）。每个框体仅能输入一个字符（字母或数字）。
  - **智能跳转**：输入单个字符后，焦点会自动跳转到下一个输入框。
  - **便捷修改**：使用回退键（Backspace）可以删除前一个框的内容并返回该框。
  - **自动验证**：当您填满所有输入框后，系统将自动发起验证请求。
- **验证判定**：
  - **成功**：验证通过，您将自动进入主搜索界面。
  - **失败**：所有输入框的边框将变为红色，提示代码无效。请仔细检查您的代码是否输入正确，或联系管理员获取有效的代码。

### 如何获取认证代码（MD5身份代码）

本系统的认证代码并非由管理员直接分配，而是需要您通过以下步骤自行生成和申请。

- **前往指定频道**：
  请前往 QQ 频道 hwl20090511。
- **设定个人密钥**：
  您需要自行设定一个 8位 的密钥（Key）。这个密钥是您个人的“原始密码”，请务必牢记。
  - **密钥要求**：必须 同时包含大写字母和数字 （例如：MYKEY123）。
- **生成MD5身份代码**：
  请使用频道首页下方提供的 MD5 小程序（或任何标准MD5生成工具），对您的8位密钥进行加密。

::: warning 加密规则（必须遵守）
- **原始密钥**：输入小程序用于计算的密钥，所有字母必须为 大写（例如 MYKEY123）。
- **生成结果**：转换后，您将得到一个32位的MD5字符串。请确保该字符串的所有字母为 小写。
  这个32位的小写MD5字符串（例如 f5d6...）就是您登录系统所需的 唯一身份代码。
:::

### 申请流程与格式

生成MD5身份代码后，您需要按以下流程提交申请以激活账户：

- **申请地点**：请 不要 在本文档下方或频道的公共聊天区评论申请。
- **正确方法**：请前往频道的 “账号申请” 专属页面（或相关子版块），创建一个新帖子。
- **重要操作**：发帖后，请务必 点赞您自己的帖子。这是通知管理员审核的唯一途径（QQ频道仅发帖不会向管理员推送通知）。
- **等待审核**：管理员会及时审核您的申请。当操作完成（代码被汇入数据库）时，我们会在您的申请帖子下方评论通知您。

#### 申请模板

请根据您的需求，严格按照以下格式发帖：

1. **普通账户申请**

   ```markdown
   [普通账户申请]
   申请昵称：（此昵称将展示于系统登录后的界面，无需实名）
   MD5：（请填写您加密后得到的32位小写MD5字符串）
   ```

2. **特殊权限申请**

   ```markdown
   [特殊权限申请]
   原MD5代码：（您当前正在使用的普通账户MD5）
   新MD5代码：（请生成一个全新的8位密钥及其对应的MD5）
   申请理由：（请详细说明您的科研或研究用途）
   申请权限项目：（请具体列出您需要的数据项，例如：“户籍地”、“精确地域查询”）
   ```

3. **账户安全申请（重置代码）**

   ```markdown
   [账户安全申请]
   原MD5代码：（您怀疑已经泄露或希望更换的MD5）
   新MD5代码：（请生成一个全新的8位密钥及其对应的MD5）
   ```

### 权限与安全说明

- **普通账户**：为了最大程度保护学生隐私，普通账户 暂时禁用 了“户籍地”和“精确地域代码”（如区县级）的查询权限。
- **特殊权限**：
  - 您必须 首先拥有一个普通账户 才能申请特殊权限。如果您没有，请发送两份独立的申请帖（一份普通，一份特殊），否则您的合并申请将被拒绝。
  - 出于安全考虑，一旦您申请特殊权限，就 必须更换您的MD5代码（即同时提交新旧两个MD5）。
  - 我们不主动公布系统隐藏的所有权限。请您根据实际需求提出申请，管理员商议后会尽可能满足，并在评论区回复您已开通的权限。
- **代码安全**：如果您觉得您的身份代码（MD5）已经泄露，请立即提交 [账户安全申请] 来轮换您的代码。

::: danger 关于MD5代码泄露的风险说明
您可能会担心，在申请时短暂暴露MD5代码是否安全？

**MD5是单向的**：
MD5是一种“摘要算法”，它只能从您的密钥（如 MYKEY123）单向计算出MD5值。它在算法上是 无法被反向破解（即从MD5值推算出您的原始密钥）的。

**暴力破解的难度**：
如果您的密钥严格遵守“8位大写字母+数字”的组合，黑客只能进行暴力破解（即“穷举”，尝试所有可能的组合）。按目前主流算力（假设每毫秒校验一次MD5值）计算，完整遍历所有可能组合的时间 预计约为89年。

**真正的风险：“撞库”**：
最大的威胁来自于 “撞库”（Dictionary Attack）。如果您设置的密钥过于简单（例如 12345678 或 ABCDEFGH），黑客会使用一个包含亿万个常用密码的“字典”来进行批量比对。一旦您的简单密钥在字典中被命中，您的身份代码就会立即泄露。

**结论**：短暂暴露MD5本身风险极低。但请务必设置一个 无规律的、包含大小写和数字的8位密钥。如果怀疑泄露，请立即更换。
:::

## 🔍 **搜索功能**

认证成功后，您将看到主搜索界面。界面顶部有两个标签页，您可以随时切换：

1. **按姓名查询 (默认)**

   此模式用于通过姓名或拼音缩写查找学生。

   - **姓名输入**：在输入框中键入学生姓名。系统支持 模糊匹配（例如，输入“张”即可检索所有姓“张”的学生）。
   - **首字母缩写**：支持 Pinyin 首字母缩写搜索。
     - 请注意：对于翘舌音（如 zh、ch、sh），必须保留 'h'（例如，应输入 ZHANG 而不是 ZANG），否则无法正确匹配。
   - **智能建议**：输入过程中，系统会实时在下方显示最多4个匹配的姓名建议。您可以点击建议项来自动填充并立即执行搜索。
   - **执行搜索**：点击“查询”按钮或在输入框中按 Enter 键。
   - **历史记录**：搜索框下方会显示您最近的10条搜索历史。点击历史项可快速重复该次搜索。点击旁边的 🗑️ 图标可清除所有历史记录（历史记录存储在您的本地浏览器中）。

2. **代码查询**

   此模式用于通过特定代码（如生日、学号、QQ号等）进行精确或批量查询。

   - **切换标签**：点击“代码查询”标签页。
   - **动态提示**：当您在输入框中输入时，系统会根据您输入的 数字长度，在输入框下方 动态提示 您当前匹配的查询类型。
   - **支持的代码类型**：
     - **生日查询 (4 / 6 / 8位数字)**
       - 4位 (年份)：如 2008，查询2008年出生的所有学生。
       - 6位 (年月)：如 0802，查询2008年2月出生的所有学生。
       - 8位 (年月日)：如 20080215，查询2008年2月15日出生的学生。
     - **地区代码 (1 - 6位数字)**
       - 1位 (行政大区)：如 3，查询华东地区学生。
       - 2位 (省份)：如 44，查询广东省学生。
       - 4位 (地级市)：如 4403，查询深圳市学生。
       - 6位 (区县)：如 440303，查询深圳市福田区学生。（注：精确地域查询需特殊权限）
     - **班级代码 (5位数字)**
       - 格式为 级数(2位) + 年级(1位) + 班级(2位)。
       - 示例：如 24105，查询2024级（级数）高一（年级）5班（班级）的学生。
     - **完整学号 (18位数字)**
       - 输入完整的18位学号进行精确查询。
     - **QQ号 (9 - 11位数字)**
       - 查询系统中录入的、匹配该QQ号的学生。
   - **执行搜索**：点击“查询”按钮或按 Enter 键。

**通用搜索提示**：
- 当您点击输入框时，系统会自动清空上一次的输入内容，方便您进行新的查询。
- 查询执行时，系统会显示“正在查询，请稍候...”的加载提示。
- 如果数据库中没有匹配项，系统将显示“未找到匹配的学生信息”。

## 📊 **查询结果**

所有查询结果将以清晰的 玻璃拟物风格卡片 呈现在搜索栏下方。结果分为单一学生和多个学生两种情况。

1. **单一学生结果**

   当查询结果为 唯一学生 时，信息将分布在多个卡片中。

   - **基本信息卡片**：
     - 姓名、性别、级数。
     - 学号、户籍地（需权限）、生日。
     - 学生类型（如走读生/寄宿生）。
     - 照片：如果该学生有可用的照片，将显示在姓名卡片右侧。
   - **详细信息卡片（以下卡片仅限特定权限用户可见）**：
     - **团员信息**：职位、入团日期、团员编号、电话、民族。
     - **社团信息**：所属社团、担任职位。
     - **QQ信息**：QQ号、昵称、个性签名、QID、等级、关联邮箱、注册时间、头像。
   - **考试成绩（仅限特定权限用户）**：
     - 默认显示该学生所有考试的 简要总结（如分科/总平均分、平均校次/班次）。
     - 点击 “详情” 按钮可展开详细数据：
       - **趋势图**：包含“校次趋势图”（多科目折线对比）和“总分&校次趋势图”。
       - **考试列表**：列出历次考试。点击任意一次考试卡片，将以详细表格形式展示该次考试的：分数、年级均分、相对比例、校次、班次。
   - **所在班级**：
     - 显示学生所属的每个班级（如行政班、选科班）。卡片上包含班级名称、班级选课组合、班级类型（如普通班/重点班/逐梦班/网校直播班）。
     - 点击 “详情” 按钮可展开该班级的详细数据：
       - 班主任、各科任课老师。
       - 同学列表：默认按生日排序。点击列表中的同学姓名，可 跳转查询 该同学的详细信息。
       - 班级统计：平均生日 & 与查询学生的差距、生日分布散点图（图表支持热区响应，鼠标悬停或点击数据点可显示当月生日的学生）。
   - **返回按钮**：如果您是从“多个学生结果”列表点击进入详情的，系统会显示一个“返回列表”按钮。

2. **多个学生结果**

   当查询命中 多名学生 时（例如按年份或班级代码查询）：

   - **统计标题**：首先显示“找到 X 个匹配的学生”的统计数据。
   - **排序功能**：您可以使用结果列表顶部的 下拉菜单，选择“按姓名”、“按学号”或“按生日”对结果进行排序。
   - **学生列表**：每个列表项清晰地显示学生的姓名和学号（学号中间部分会做隐私隐藏）。
   - **查看详情**：点击任意学生项右侧的“详情”按钮，即可进入该学生的“单一学生结果”页面。

3. **无结果或错误**

   - **无结果**：如上所述，显示“未找到匹配的学生信息”。
   - **查询错误**：如果您试图查询一个被系统禁止查询的学生（出于隐私保护），系统将显示一条红色的错误消息。

## 🛠️ **其他功能**

- **查询提示**：在您进行首次查询前，界面会显示“请输入姓名”以及支持的查询代码示例列表。
- **响应式设计**：系统界面完美适配手机、平板等移动设备。在小屏幕上，部分图表（如成绩趋势图）支持横向滚动查看。
- **隐私保护**：敏感信息（如电话、完整学号）会根据您的认证代码权限，自动进行隐藏或部分（脱敏）显示。
- **历史管理**：搜索历史记录存储在您的本地浏览器缓存中，点击 🗑️ 图标即可清除。

## ⚠️ **注意事项**

::: warning 重要注意事项
- **权限限制**：您的认证代码 直接决定 您的数据可见范围。如果您发现某些信息（如考试成绩、电话号码）不可见，这代表您的代码权限不足。如确有工作需要，请按上文流程提交 [特殊权限申请]。
- **数据准确性**：本系统数据来源于学校最新的CSV数据，但仍可能存在一定的更新延迟。生日等数据可能在录入时有误，后续会进行修正。
- **禁止查询**：部分学生的敏感信息已被系统锁定保护。如果您在同学列表中点击某位同学无响应或查询时提示错误，即代表该学生信息禁止查询。
- **查询性能**：当您执行大范围查询时（例如查询整个省份或整个年级的学生），系统需要处理大量数据，查询时间可能稍慢（几秒到十几秒不等），请耐心等待加载。
- **反馈渠道**：如您在使用中遇到Bug或有功能建议，请通过文档链接或指定渠道进行反馈。
:::
